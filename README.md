# Frog Flight Mini App

Полный контур для Telegram Mini App и Telegram-бота:

- игроки авторизуются по `Telegram ID`
- backend хранит пользователей, рейсы, рекорды и попытки
- хозяйка заводит рейс через бота и получает уведомления
- админ имеет отдельные внутренние команды

## Файлы

- `index.html`, `style.css`, `script.js` — клиент Mini App
- `server.js` — HTTP API, SQLite, бот, уведомления
- `data/app.sqlite` — база данных, создаётся автоматически

## Переменные окружения

Скопируй `.env.example` в `.env` и заполни:

```env
PORT=3000
DATA_DIR=./data
PUBLIC_URL=https://your-domain.example
TELEGRAM_BOT_TOKEN=123456:replace_me
DISABLE_TELEGRAM_BOT=0
OWNER_TELEGRAM_ID=123456789
ADMIN_TELEGRAM_IDS=111111111,222222222
REMINDER_BEFORE_END_MINUTES=30
```

Что важно:

- `PUBLIC_URL` должен быть `https://...`
- `DATA_DIR` — папка для SQLite; на Railway удобно ставить `/data`
- `OWNER_TELEGRAM_ID` — Telegram ID хозяйки
- `ADMIN_TELEGRAM_IDS` — список Telegram ID админов через запятую
- `DISABLE_TELEGRAM_BOT=1` можно включить локально для отладки API без polling и вызовов в Telegram

## Запуск

Требуется `Node.js 22.x`.

```bash
npm start
```

Сервер поднимется на `http://localhost:3000`.

## Как работает авторизация

- Mini App отправляет `initData` на backend
- backend валидирует подпись через токен бота
- пользователь сохраняется по `telegram_id`
- рекорды и разблокировка маршрута тоже привязаны к `telegram_id`

Если backend запущен с токеном бота, локальная браузерная регистрация больше не считается рабочим входом: играть и писать рекорды можно только из Telegram Mini App.

## Роли

### Игрок

- открывает Mini App
- играет под своим Telegram-профилем
- попадает в общий рейтинг рейса

### Хозяйка

- создаёт рейс через бота
- переводит страницу в режим отдыха
- получает уведомления о старте, напоминание перед концом и итоговую сводку после завершения рейса

### Админ

- имеет все действия хозяйки
- может смотреть внутреннюю сводку
- может менять время напоминания

## Команды бота

Доступно всем:

- `/start`
- `/help`
- `/status`
- `/whoami`

Для хозяйки и админа:

- `/setflight` — пошагово создать новый рейс
- `/setrest` — перевести страницу в режим отдыха
- `/notifytest` — проверить личные уведомления
- `/cancel` — отменить текущий мастер

Только для админа:

- `/adminstats`
- `/setreminder 30`

## Что делает backend

- хранит пользователей, рейсы, рекорды и все попытки
- сортирует рейтинг по правилу: выше тот, кто набрал раньше при равном счёте
- выключает отправку очков после окончания рейса
- рассылает уведомления по расписанию
- отдаёт клиенту текущее состояние рейса и таблицу лидеров

## Продакшен

Для реального запуска нужен любой хостинг с `Node.js` и `HTTPS`.

Минимальный порядок:

1. Развернуть проект на сервере.
2. Заполнить `.env`.
3. Указать `PUBLIC_URL`.
4. В `@BotFather` настроить Mini App / menu button на этот URL.
5. Запустить бота и открыть ему личный чат хотя бы один раз с аккаунта хозяйки и админа, чтобы бот сохранил их `chat_id`.
